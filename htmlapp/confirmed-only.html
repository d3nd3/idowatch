All buttons send exact TX from logcat_dumps/*.txt. Categories: GET (0x02), SET (0x03), BIND (0x04), MSG (0x05), Control (0x06), V3 (0x33…, health sync, activity). Weather is SET 0x2D.

Open DevTools (F12) → Console to see parsed interpretations of each RX packet. Confirm or correct the guesses there.

GET (cmd 0x02)

Read state / info. First byte 0x02; second byte is the key. Keys in ascending order.

SET (cmd 0x03)

Write config. First byte 0x03; second byte is the key. Keys in ascending order (0x01 → 0x60).

0x32 Brightness / screen

Model: IDOSetScreenBrightnessInfoBluetoothModel. Packet: 03 32 [levelValue] [isManual] [mode] [autoAdjustNight] [startHour] [startMinute] [endHour] [endMinute] [nightLevel] [showInterval] (bytes 2–11). Same as MainSetActivity.setScreenBrightness(level) / BLEManager.setScreenBrightness(int): Day level = level (1–100). See Set screen brightness in IDOGitBook and GET 0x02 0x32 parser below.

0x35 Connection parameters (VBUS_EVT_APP_SET_CONN_PARAM)

VeryFit uses a two-step SET 0x35 handshake around heavy operations like v3 health sync and watch-face writes. The SDK’s IDOSetConnectParam describes a JSON model (mode, max_interval, etc.), but on this firmware the wire-level packets are simple 12-byte frames where byte2 is the conn-param step: 03 35 00..., 03 35 01..., 03 35 02.... The UI below replays the exact TX from set_conn_param.txt and ui_watch_face_write_json.txt; replies are parsed as generic SET ACKs.

0x02 Alarms (legacy) + V3 alarms

Legacy SET 0x03 0x02 per-alarm payload is: [alarm_id][status][type][hour][minute][repeat][snooze_min]. V3 alarms (cmd 0x0E / VBUS_EVT_FUNC_V3_SPORT_SORT) send a single frame for all 10 slots; the wire payload in set_alarms_and_sports.txt is: [maxAlarms=0x0A][AlarmIndex][on/off][hour][minute][enabled+repeat bits][snoozeMinutes_L][snoozeMinutes_H][snoozeFreq_L][snoozeFreq_H][24B alarmName]… repeated for each alarm index. The UI below manages a 10-slot list and maps the form into that richer V3 structure while keeping the simpler legacy 0x02 helper available.

0x30 Notice / call alert (VBUS_EVT_APP_SET_NOTICE) — not fully understood

20-byte payload. app_fresh_launch.txt: 03 30 88 00 00 AA 00… = enable; 03 30 88 00 00 55 00… = disable (0x55=off, same as Music/DND).

Angelfit 5-byte payload. notify_switch and call_switch: 0→0x55, 1→0x88. notify_itme1, notify_itme2, call_delay: raw 0–255.

BIND (cmd 0x04)

VBUS_EVT_APP_BIND_START (evt 200). app_fresh_launch sends this after GET MTU; watch replies to confirm bind. First byte 0x04.

MSG (cmd 0x05)

Phone → watch notifications. 0x05 0x01 = call (KEY_MSG_CALL), 0x05 0x02 = call status (KEY_MSG_CALL_STATUS), 0x05 0x03 = MSG (KEY_MSG_MSG). Chunked in 16-byte payloads for firmware format. TOOBUR.md; angelfit protocol.h.

0x05 0x02 Call status

0x05 0x01 / 0x05 0x03 — send custom (form)

Sender = caller name (call) or message sender. Message = body. For Firmware MSG (0x05 0x03), first payload byte = source app type (0–255). 0=nothing (invalid, no notification sent), 1=SMS, 2=Mail, 3=WeChat, 4=Snapchat, 5=invalid, 6=Facebook, 7=Twitter, 8=WhatsApp, 9=Facebook Messenger, 10=Instagram, 11=LinkedIn, 12=Calendar, 13=Skype.

Control (cmd 0x06)

Control commands are watch → phone actions (the watch sends to the phone). VBUS_EVT_APP_APP_TO_BLE_*: Music (500/501), Photo (APP_TO_BLE_PHOTO_*), Find device (FIND_DEVICE_*), Single sport, ANCS. TOOBUR.md table; index.html.

Other (cmd 0xF0)

RESET 0xF0 0x01 = Reboot. Watch reboots; connection drops. TOOBUR.md.

V3 (cmd 0x33 — 33 DA AD DA AD…)

Extended protocol. Sub-command in bytes 4–5 (e.g. 0x0B 0x06 = watch face select). Ordered ascending by sub-cmd.

v3 0B 06 / 0B 07 — Watch face

v3 0C 0F — GET alarm (VBUS_EVT_FUNC_V3_GET_ALARM evt 5018)

Request alarm list from watch. Reply is multi-packet (353+2 CRC bytes); parsed alarms are loaded into the alarm list below. See logcat_dumps/get_alarm.txt.

v3 0x0E — Set sport type sort (setSportSortV3 / VBUS_EVT_FUNC_V3_SPORT_SORT)

Function table: getSportsTypeV3. Event: base_app_set + func_v3_sport_sort. JSON: num (1–30), item array of { index, type } — index from 1, type = sport type id. Same v3 cmd 0x0E as in set_alarms_and_sports.txt (evt 5017).

v3 health sync (get_sync_health_v3.txt)

Get sizes = v3 cmd 0x05: ask the watch how much health data exists. Then use cmd 0x04 per data type: 4 = workout/activity records, 7 = sleep, 8 = daily sport summary (steps / kcal / distance). On this firmware, type 8 is the useful path for day totals; type 7 for sleep data.

v3 1A xx — Function table (raw)

Sends the captured 0x33 0x1A frame from app_fresh_launch.txt (cmd id 0x1A, seq 0x02) on devices where func table bit v3_function_table_33_1a is set. Reply will be tagged as a v3 function-table packet in the console.
